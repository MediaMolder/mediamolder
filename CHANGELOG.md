# Changelog

All notable changes to MediaMolder are documented in this file.
Format follows [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Added
- **Stream-copy nodes (`type: "copy"`).** A new node category lets a graph forward demuxer `AVPacket`s straight to the muxer with no decode/encode — the standard "swap container, keep streams" pattern. One copy node per media type (`copy_video`, `copy_audio`, `copy_subtitle`, `copy_data`) is exposed under the new **Copy** palette section between Encoders and Processors. The implicit-encoder splice treats copy nodes as already-encoded, so a manual `in:a:0 → copy_audio → out:a` edge no longer gets an AAC encoder spliced in. The runtime adds the output stream via `avcodec_parameters_copy` (no encoder context required), rescales packet timestamps from the demuxer's `time_base` to the muxer's post-header `time_base`, and clones packets when one demuxed stream feeds multiple copy nodes. Implemented across `graph/graph.go` (new `KindCopy`), `av/packet.go` (`Packet.Rescale`, `ClonePacket`), `av/mux.go` (`AddStreamFromInput`, `StreamTimeBase`), `pipeline/handlers.go` (new `handleCopy` + per-stream-type packet routing in `handleSource` + per-channel rescale in `handleSink`), `internal/gui/api.go` (palette entries + new `"Copy"` category in `categoryOrder`), and `frontend/src/lib/spawn.ts` / `MMNode.tsx` / `jsonAdapter.ts`. Documented in [docs/json-config-reference.md](docs/json-config-reference.md) and [docs/gui.md](docs/gui.md).
- **ABR ladder example** (`testdata/examples/35_abr_ladder.json`) — single input fans out to four `scale` filters (1920×1080, 1280×720, 960×540, 640×360), each feeds a dedicated `libx264` encoder at 7000/4000/2000/1000 kbps, with a single AAC audio encoder fanned out to all four MP4 muxers. Exercises explicit fan-out, multi-output mux, and per-encoder bitrate independence end-to-end.

### Changed
- **Schema v1.2 is now accepted by the runtime validator.** `pipeline.Config.validate` previously hard-rejected `"1.2"` even though the GUI emits it for every newly-saved job. v1.2 currently adds nothing to the schema beyond `graph.ui.positions` (already accepted) and is forward-compatible with v1.0/v1.1.
- **Encoder parameters are forwarded verbatim to libavcodec.** Every key in a node's `params` that is not a runtime-reserved field (`codec`, `width`, `height`, `bitrate`, `threads`, `thread_type`) is now passed through `EncoderOptions.ExtraOpts → av_dict_set → avcodec_open2`, so codec-specific options written by the GUI (`preset`, `crf`, `maxrate`, `bufsize`, `x264-params`, …) actually reach the encoder. `b` is recognised as an alias for `bitrate`, and `g` maps to `EncoderOptions.GOPSize`. Covered by `TestCollectEncoderExtraOpts` in `pipeline/handlers_test.go`.
- **GUI: ABR audio encoded once and fanned out.** `35_abr_ladder.json` now defines a single explicit `enc_audio` (codec `aac`, 128 kbps) with edges to all four output muxers, instead of four redundant implicit AAC encoders.
- **GUI: `libx264` / `libx265` `preset` rendered as a dropdown** with the canonical 10 entries (`ultrafast` … `placebo`, default `medium`). The structure is generic — add to `ENCODER_OPTION_CHOICES` in `frontend/src/lib/encoderSchema.ts` to surface other codec presets the same way (e.g. libvpx_vp9 `deadline`).

### Fixed
- **GUI: CRF / QP defaults no longer show as `-1`.** libavcodec encodes "unset, pick the codec's built-in default" as the sentinel `(default=-1, min=-1, max=FLT_MAX)` for CRF/QP-style options on libx264/libx265/libsvtav1/libvpx_vp9/libaom_av1. The Inspector now substitutes a per-codec override table (`ENCODER_PRIMARY_OVERRIDES`) so the placeholder reads `(default — 23)` and the slider/number bounds are realistic (e.g. libx265 CRF 0–51 with default 28).
- **GUI: edges serialise using the node id, not the display label.** `deriveEndpoint` was returning `node.data.label` for graph nodes, which is the codec/filter display name (e.g. `aac`, `libx264`); a hand-wired `enc_audio` was being saved as `to: "aac"` and the runtime then failed with `unknown destination node aac`. The graph-node branch now emits the React Flow id (which equals the job-side id).

### Added (earlier in this cycle)
- **Encoder Inspector — Advanced view, search, and Raw options.** The Inspector's encoder form now exposes every remaining AVOption under a collapsible **Advanced** section, bucketed by a lightweight heuristic (`Threading`, `Quality`, `Color`, `Motion`, `Profile / Level`, `GOP & frames`, `Other`). A search box filters across all groups by option name and help text. A separate **Raw options** section surfaces codec-native parameter strings (`x264-params`, `x264-opts`, `x265-params`, `svtav1-params`, `aom-params`, `vpx-params`) as multi-line text inputs that are passed through verbatim. Completes PR3 of 3 for the structured encoder configuration UI. Implemented entirely in `frontend/src/components/EncoderForm.tsx` (no new backend dependencies).
- **Encoder Inspector form (primary controls).** Selecting an encoder node now loads the codec's option schema from `/api/encoders/{name}/options` and renders typed controls for the four most common roles — Preset, Rate control, Quality (CRF/CQ/Q) **or** Bit rate (whichever applies), and Keyframe interval — using the codec's own ranges, defaults, and named values (`AV_OPT_TYPE_CONST` enums become dropdowns). Extra options can still be set under the existing key/value Params editor; the upcoming PR3 will add the full Advanced view, search, and raw-options escape hatches. Implemented in `frontend/src/lib/encoderSchema.ts` (typed fetcher + per-encoder UI-role table for libx264/libx265/libsvtav1/libvpx_vp9/libaom_av1/h264_nvenc/hevc_nvenc/videotoolbox/aac/libfdk_aac/libopus/libmp3lame), `frontend/src/components/controls/OptionControl.tsx`, and `frontend/src/components/EncoderForm.tsx`. Also fixed `frontend/src/lib/spawn.ts` so dragging an encoder onto the canvas now stores its codec name in the canonical `params.codec` slot (was previously placed in the unused `filter` field).
- **Encoder option enumeration API (`GET /api/encoders/{name}/options`)** — backend endpoint that walks both the generic `AVCodecContext` class and the codec's `priv_class` via `av_opt_next`, returning every AVOption with type, range, default, and named constants (`AV_OPT_TYPE_CONST` children grouped by `unit`). Foundation for the upcoming structured Inspector form for encoder nodes (PR1 of 3). Implemented in `av/options.go` (cgo helpers + `EncoderInfo`/`EncoderOption`/`EncoderOptionEnum` types) and `internal/gui/encoders.go` (handler with in-memory caching).
- **GUI: edges can now be selected, deleted, and re-routed.** Clicking a connection selects it (drawn thicker with a glow), and `Backspace`/`Delete` removes the selected edge(s). The hover popover gained a **Delete** button. The endpoint of any existing connection can be grabbed and dragged to a different handle to re-route. Documented in `docs/gui.md` and the in-app Help dialog. Implemented in `frontend/src/app.tsx` (selectedEdgeIds + Backspace handler), `frontend/src/components/MMEdge.tsx` (Delete button via `useReactFlow().deleteElements`), and `frontend/src/styles.css` (.react-flow__edge.selected highlight).
- **GUI: edge attribute popover with full technical metadata** — edges no longer carry an inline text label (which cluttered dense graphs). Hover (or click to pin) any connection to open a popover at the midpoint listing every known attribute for the stream and which node established each value. The probe + inference pipeline now surfaces a much richer set of properties: codec profile/level, bit_rate, bit_depth, color_space, color_range, color_primaries, color_transfer, sample aspect ratio, field order, r_frame_rate, codec FourCC tag, and start time, in addition to the existing width/height/pix_fmt/frame_rate/sample_rate/sample_fmt/channels/channel_layout/duration. Image files are reported as a single video stream with the image's geometry and pixel format. Implemented in `frontend/src/components/MMEdge.tsx`, `frontend/src/lib/streamAttrs.ts`, `av/demux.go`, `av/probe_names.go` (new helpers `ColorSpaceName`, `ColorRangeName`, `ColorPrimariesName`, `ColorTransferName`, `ProfileName`, `FieldOrderName`, `CodecTagString`), and `internal/gui/probe.go`.
- **GUI: `Get properties` button on Input nodes** — clicking it in the Inspector calls a new `POST /api/probe` backend endpoint that opens the source URL with `avformat_open_input` + `avformat_find_stream_info` and returns per-stream technical metadata (codec, width/height, pix_fmt, frame rate, sample rate, sample format, channels, channel layout, duration). The probed values are stored on the Input node (editor-only — never serialised) and seed the edge attribute inference, so every downstream connection in the graph gets accurate technical attributes automatically. Implemented in `internal/gui/probe.go`, `av/probe_names.go` (helpers `PixFmtName`, `SampleFmtName`, `CodecName`, `DefaultChannelLayoutName`), and `frontend/src/components/Inspector.tsx`.
- **Visual editor (`mediamolder gui` subcommand)** — browser-based pipeline editor served from the same single binary as the CLI. Drag-and-drop palette populated from `/api/nodes` (every libavfilter, codec, demuxer/muxer, and registered Go processor in the binary), stream-typed handles and edges, dagre auto-layout, JSON import/export, and a typed inspector for every node kind.
- **Live run + progress streaming** — Run/Stop buttons execute the current graph via `POST /api/run`; per-job state, metrics, errors, and logs stream back over Server-Sent Events (`GET /api/events/{jobId}`). Live frame counts, FPS, and error highlights overlay each node on the canvas.
- **GUI HTTP API**: `/api/health`, `/api/nodes`, `/api/examples`, `/api/validate`, `/api/run`, `/api/cancel/{jobId}`, `/api/events/{jobId}`.
- **Schema v1.2** adds optional `graph.ui.positions` for editor-side node-position persistence. Runtime ignores the block; older v1.0 / v1.1 jobs load unchanged.
- `internal/gui` package: embedded production frontend via `//go:embed`, job manager with bounded history replay (64 events) and finished-job retention (16 runs).
- `frontend/` workspace: Vite 6 + React 19 + TypeScript strict + @xyflow/react v12 + dagre + Zustand.
- Makefile targets `frontend-install`, `frontend-dev`, `frontend-build`, `gui`, `gui-dev`, `build-gui`.
- `docs/gui.md` — full GUI user + developer guide.
- CI: `gui` job builds the frontend and the embedded GUI binary on every push.
- **`go_processor` node type** — custom Go per-frame processing in the MediaMolder graph (AI inference, analytics, tracking, metadata emission). Requires `schema_version: "1.1"`.
- `processors` package with `Processor` interface (`Init`/`Process`/`Close`), thread-safe registry, `ProcessorContext`, and `Metadata`/`Detection` types.
- Built-in processors: `null` (passthrough), `frame_counter` (counting with periodic metadata), `frame_info` (frame dimensions/format/PTS diagnostics), `scene_change` (scene detection using the same MAFD + diff-of-MAFD algorithm as FFmpeg's `scdet` filter — zero-copy Y plane access for YUV formats, swscale GRAY8 fallback for RGB).
- `av.Frame.Clone()` — reference-counted frame clone via `av_frame_clone()`.
- `av.FrameSceneScore(a, b)` — computes luma MAFD between two frames (0–100 scale). For YUV planar formats, reads the Y plane directly with zero allocation; falls back to swscale GRAY8 conversion for RGB/packed formats.
- Helper functions in `processors/helpers.go`: `Letterbox`, `ImageToFloat32Tensor`, `DrawDetections`, `FrameToRGBA`, and `FrameToFloat32Tensor`.
- `av.Frame.ToRGBA()` — converts any video frame to `*image.RGBA` via libswscale (supports YUV420P, NV12, RGB24, and all other FFmpeg pixel formats). Also adds `Frame.PixFmt()` accessor.
- `av.Frame.PixFmt()` — returns the frame's pixel format as an `int` (`AVPixelFormat`).
- **Optional `yolo_v8` built-in processor** (behind `-tags with_onnx`): YOLOv8 object detection via ONNX Runtime with CUDA support, greedy NMS, and letterbox-aware coordinate mapping. Pure-Go post-processing (`ParseYOLOv8Output`, `NMS`, `IoU`) in `processors/yolov8.go` compiles without ONNX.
- `docs/yolov8-guide.md` — end-to-end guide for setting up and using the YOLOv8 processor.
- **`metadata_file_writer` built-in processor** — decorator that wraps any processor and writes emitted metadata to a JSON Lines file. Configurable entirely in JSON with `output_file` and `inner_processor` params.
- **`--metadata-out` CLI flag** on `mediamolder run` — writes all `ProcessorMetadata` events as JSON Lines to a file (or stdout with `-`).
- JSON tags on `pipeline.ProcessorMetadata` for clean serialisation (`node_id`, `frame_index`, `pts`, `metadata`).
- Schema v1.1 (`schema/v1.1.json`) adding `go_processor` to node type enum with conditional `processor` field requirement.
- `mediamolder list-processors` CLI command.
- Comprehensive documentation: `docs/go-processor-nodes.md`, updated `docs/json-config-reference.md` and `README.md`.

### Changed
- **GUI: stream-type legend moved to the bottom centre** and laid out horizontally so it no longer overlays the bottom-right minimap.
- **GUI: toolbar Help button is now labelled `Help`** instead of `?` (the `?` keyboard shortcut still opens the dialog).
- Graph builder and runtime support `go_processor` nodes with identical edge/pad semantics as filters.
- Backward-compatible: all existing `schema_version: "1.0"` pipelines work unchanged.
